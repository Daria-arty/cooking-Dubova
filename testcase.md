Тест-кейсы
TC-01. Успешная запись с последующим ручным закрытием модалки
ID: TC-01
Заголовок: Проверка, что ручное закрытие модалки после успеха не влияет на последующие модалки
Предусловия: Приложение загружено, список слотов отображается.
Шаги:

    Кликнуть по карточке «Итальянская кухня для новичков».
    Ввести имя: Иван, телефон: +7 900 123 45 67.
    Нажать «Записаться».
    Дождаться появления зелёного сообщения «Вы успешно записаны…».
    Сразу же (не дожидаясь 2 секунд) кликнуть по крестику × для закрытия модалки.
    Кликнуть по карточке «Азиатская кухня: суши и роллы».
    Подождать 2 секунды.

Ожидаемый результат: Новая модалка остаётся открытой, не закрывается автоматически. Поля формы пусты, фокус на поле «Имя».
TC-02. Прокрутка страницы при открытой модалке
ID: TC-02
Заголовок: Проверка блокировки прокрутки фона при открытой модалке
Предусловия: Приложение загружено на мобильном устройстве (или в DevTools с эмуляцией мобильного).
Шаги:

    Прокрутить страницу вниз, чтобы карточки расписания были видны.
    Кликнуть по любой карточке слота.
    Попробовать прокрутить страницу пальцем (или мышью) за пределами модалки.
    Закрыть модалку кликом по overlay.
    Попробовать прокрутить страницу.

Ожидаемый результат: На шаге 3 страница не прокручивается — фон заблокирован. На шаге 5 прокрутка снова работает.
TC-03. Повторная отправка формы при медленной сети
ID: TC-03
Заголовок: Проверка защиты от дублирования запросов при быстром повторном нажатии
Предусловия: В коде createBooking увеличить setTimeout до 5000 мс (или эмулировать медленную сеть в DevTools).
Шаги:

    Открыть модалку для записи.
    Ввести валидные данные: имя Иван, телефон +7 900 123 45 67.
    Быстро нажать кнопку «Записаться» 3 раза подряд (или нажать Enter в поле ввода 3 раза).
    Подождать 5 секунд.

Ожидаемый результат: Отправляется только один запрос к API. Появляется одно сообщение об успехе. Кнопка после первого клика переходит в состояние «Отправка…» и блокируется, повторные клики игнорируются.
TC-04. Валидация имени со спецсимволами
ID: TC-04
Заголовок: Проверка, что имя с цифрами и спецсимволами отклоняется
Предусловия: Модалка открыта.
Шаги:

    В поле «Имя» ввести 12345.
    В поле «Телефон» ввести +7 900 123 45 67.
    Нажать «Записаться».
    Очистить поле «Имя» и ввести !@#$%^&*().
    Нажать «Записаться».
    Очистить поле «Имя» и ввести Иван123.
    Нажать «Записаться».

Ожидаемый результат: На шагах 3, 5, 7 появляется ошибка «Имя может содержать только буквы, пробелы и дефисы». Форма не отправляется.

ID: TC-05
Заголовок: Проверка, что имя из одних пробелов отклоняется
Предусловия: Модалка открыта.

Ожидаемый результат: Появляется ошибка «Имя обязательно для заполнения». Форма не отправляется.
TC-06. Обработка ошибки 409 с сохранением данных формы
ID: TC-06
Заголовок: Проверка, что при ошибке 409 данные формы не теряются
Предусловия: Модалка открыта.
Шаги:

    Ввести имя: Тест 409, телефон: +7 900 123 45 67.
    Отметить чекбокс «Нужен фартук».
    В поле «Аллергии» ввести Орехи.
    Нажать «Записаться».
    Дождаться появления красного сообщения «Вы уже записаны на это занятие».
    Проверить состояние полей формы.

Ожидаемый результат: Все введённые данные (имя, телефон, чекбокс, аллергии) сохраняются в форме. Пользователь может исправить имя и повторить попытку без повторного ввода остальных данных.
TC-07. Закрытие модалки по Escape
ID: TC-07
Заголовок: Проверка закрытия модалки клавишей Escape
Предусловия: Модалка открыта, фокус на одном из полей формы.
Шаги:

    Нажать клавишу Escape на клавиатуре.
    Проверить состояние модалки.
    Открыть модалку снова.
    Нажать Escape снова.

Ожидаемый результат: На шаге 1 модалка закрывается. На шаге 4 модалка снова закрывается. При повторном открытии поля формы пусты.
TC-08. Адаптивность на очень маленьком экране (320px)
ID: TC-08
Заголовок: Проверка, что модалка не выходит за границы экрана на узких устройствах
Предусловия: DevTools с шириной viewport 320px (iPhone SE).
Шаги:

    Открыть приложение.
    Кликнуть по любой карточке слота.
    Проверить отображение модалки.
    Ввести длинные данные в поле «Аллергии» (50+ символов).
    Проверить, что форма не выходит за границы экрана.

Ожидаемый результат: Модалка занимает всю ширину экрана с отступами. Кнопки «Закрыть» и «Записаться» расположены вертикально (в столбик). Все поля формы видны и доступны для ввода. Скролл внутри модалки работает корректно.
