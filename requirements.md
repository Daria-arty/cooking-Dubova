## 5. Use Cases (счастливый путь)

### UC-001: Просмотр расписания и выбор слота

**Связь:** US-001, US-002 | **FR:** FR-010–FR-017 | **Актор:** Клиент  

**Предусловия:** Клиент авторизован (FR-001); есть сетевое соединение; API доступен.  

**Постусловия:** Клиент видит список слотов или empty state; при выборе — экран деталей слота.

| Шаг | Действие актора | Отклик системы |

|-----|-----------------|----------------|

| 1 | Клиент открывает мобильное веб-приложение (главный экран / «Расписание»). | Приложение проверяет наличие сессии; при отсутствии — перенаправляет на вход (UC-000, вне данного UC). |

| 2 | — | Приложение отправляет `GET /slots?dateFrom=<сегодня>&dateTo=<сегодня+7д>` (R-027). Отображается skeleton-loader (NFR-002). |

| 3 | — | API возвращает список слотов с полями: программа, меню, дата/время начала, шеф, `availableSeats`, длительность, адрес (R-015). |

| 4 | Клиент просматривает список карточек. | Каждая карточка показывает название программы, дату/время, шефа, «Свободно: N мест»; при `availableSeats=0` — badge «Мест нет», CTA неактивен (FR-016). Слоты, отменённые студией, в выборку не попадают (FR-015). |

| 5 | Клиент нажимает на карточку слота с `availableSeats > 0`. | Открывается экран деталей: полное меню, шеф, время начала, длительность ~3 ч, адрес студии, тариф проката и `freeRentalStock` (FR-014, FR-023). |

| 6 | Клиент нажимает «Записаться». | Переход к UC-002 (создание брони). |

**Альтернатива (empty state):** на шаге 3 API возвращает пустой массив → приложение показывает «Пока нет доступных классов» (FR-011, R-027).

### UC-002: Создание брони (счастливый путь)

**Связь:** US-003, US-009, US-012 | **FR:** FR-020–FR-026, FR-024, FR-025  

**Актор:** Клиент  

**Предусловия:** Клиент на экране деталей слота; `availableSeats > 0`; слот не отменён студией.  

**Постусловия:** Создана бронь в статусе «Активна»; клиент видит экран подтверждения; счётчик мест на слоте уменьшен на бэкенде.

| Шаг | Действие актора | Отклик системы |

|-----|-----------------|----------------|

| 1 | Клиент нажимает «Записаться» на экране деталей слота. | Открывается форма бронирования: блок информации о слоте (read-only); обязательный выбор экипировки; необязательное поле «Аллергии» (FR-022, FR-024). |

| 2 | Клиент выбирает **«Своя экипировка»**. | Вариант «Прокат» не выбран; тариф проката не применяется. |

| 3 | Клиент optionally вводит текст аллергий «Орехи, лактоза». | Поле принимает произвольный текст. |

| 4 | Клиент нажимает «Подтвердить запись». | Кнопка переходит в состояние loading; отправляется `POST /bookings` с `{ slotId, equipment: "own", allergies: "..." }`. |

| 5 | — | Бэкенд атомарно проверяет места (R-004) и возвращает `201 Created` с объектом брони `bookingId`, статус «Активна»). |

| 6 | — | Приложение показывает **экран подтверждения**: дата, программа, шеф, «Своя экипировка», адрес; блок «Оплата: при посещении или переводом» (FR-025, BR-011). Кнопки: «Мои записи», «На главную». |

| 7 | Клиент нажимает «Мои записи». | Переход к UC-004; новая бронь отображается в списке активных (FR-030). |

**Вариант UC-002a (прокат):** на шаге 2 клиент выбирает **«Прокат»** → приложение показывает тариф из API и «Доступно наборов: N»; при `freeRentalStock > 0` выбор доступен; в POST уходит `equipment: "rental"`.
